## 1. Overview

Viecz is distributed via a self-hosted F-Droid repository. The F-Droid client connects over HTTPS to download the repo index and APKs. The repo is served by nginx behind a Cloudflare Tunnel, eliminating the need for direct port exposure or manual TLS certificates.

**Anti-features:** The app is tagged with `NonFreeNet` because it requires connection to `viecz-api.fishcmus.io.vn`. This is standard for client-server apps on F-Droid.

## 2. Architecture

```
F-Droid Client
      │
      │ HTTPS (443)
      ▼
Cloudflare Edge (TLS termination)
      │
      │ cloudflared tunnel (QUIC)
      ▼
cloudflared daemon → localhost:8083
      │
      ▼
nginx (port 8083) → /var/www/fdroid/repo/
```

| Component     | Role                          |
|---------------|-------------------------------|
| fdroidserver  | Generates signed repo index   |
| nginx         | Serves static repo files      |
| cloudflared   | Tunnel to Cloudflare edge     |
| Cloudflare    | DNS, TLS, CDN                 |

## 4. App Preparation

### 4.1 Fastlane Metadata

F-Droid reads app descriptions from the fastlane directory structure:

```
android/fastlane/metadata/android/en-US/
├── title.txt                 "Viecz"
├── short_description.txt     Max 80 chars
├── full_description.txt      Max 4000 chars
├── changelogs/
│   └── 1.txt                 Matches versionCode 1
└── images/
    └── icon.png              512x512 from brand/logo/mascot-512.png
```

Changelog filenames must match the `versionCode` in `build.gradle.kts` (e.g., `1.txt` for `versionCode = 1`).

### 4.2 Build Signed Release APK

```bash
cd android
./gradlew assembleRelease
```

Output: `android/app/build/outputs/apk/release/app-release.apk`

The build uses the keystore configured in `android/keystore.properties` (not committed to git). Release builds have `isMinifyEnabled = true` and `isShrinkResources = true` via R8.

### 4.3 Verify APK Signature

```bash
$ANDROID_HOME/build-tools/36.0.0/apksigner verify --print-certs \
    android/app/build/outputs/apk/release/app-release.apk
```

Expected output:

```
Signer #1 certificate DN: CN=Viecz, OU=Development, O=Viecz, L=HCMC, ST=HCMC, C=VN
```

## 5. fdroidserver Installation

The Ubuntu apt package (`fdroidserver 2.2.1`) ships with an old androguard that cannot parse R8-optimized APKs (crashes with `ResParserError: res1 must be zero!`). Use a Python venv with the latest version instead.

```bash
# Install via uv into a venv inside the repo directory
uv venv /var/www/fdroid/.venv
VIRTUAL_ENV=/var/www/fdroid/.venv uv pip install fdroidserver
```

All `fdroid` commands must use the venv binary:

```bash
/var/www/fdroid/.venv/bin/fdroid <command>
```

Current version: **fdroidserver 2.4.3**

> **Note:** The system apt package `fdroidserver` may also be installed but should NOT be used. Always use the venv binary.

## 6. Repository Initialization

### 6.1 Create Repo Directory

```bash
sudo mkdir -p /var/www/fdroid
sudo chown ubuntu:ubuntu /var/www/fdroid
```

### 6.2 Initialize

```bash
cd /var/www/fdroid
fdroid init
```

This generates:
- `config.yml` — repo configuration
- `keystore.p12` — signing keystore for repo index
- `repo/` — directory for APKs and generated index

### 6.3 Configure `config.yml`

Key settings in `/var/www/fdroid/config.yml`:

```yaml
repo_url: https://fdroid.fishcmus.io.vn/repo
repo_name: Viecz F-Droid Repo
repo_description: >-
  Official F-Droid repository for Viecz - a student freelance task marketplace.
  Apps are signed release builds from the Viecz project.

repo_keyalias: ns547299.ip-139-99-69.net
keystore: keystore.p12
keystorepass: <auto-generated>
keypass: <auto-generated>
keydname: CN=ns547299.ip-139-99-69.net, OU=F-Droid
```

The keystore passwords are auto-generated during `fdroid init` and stored in `config.yml`. Back up both `config.yml` and `keystore.p12` — losing them means clients cannot verify repo updates.

## 8. Nginx Configuration

File: `/etc/nginx/sites-available/fdroid`

```nginx
server {
    listen 127.0.0.1:8083;
    server_name fdroid.fishcmus.io.vn;

    root /var/www/fdroid;
    index index.html;

    location / {
        autoindex on;
        try_files $uri $uri/ =404;
    }

    # Proper MIME types for F-Droid repo files
    location ~* \.(jar|apk)$ {
        add_header Content-Type application/octet-stream;
    }

    location ~* \.(json)$ {
        add_header Content-Type application/json;
    }
}
```

Enable and reload:

```bash
sudo ln -sf /etc/nginx/sites-available/fdroid /etc/nginx/sites-enabled/fdroid
sudo nginx -t
sudo systemctl reload nginx
```

Nginx listens on port **8083** (not 80/443). TLS is handled by Cloudflare Tunnel.

## 9. Cloudflared Tunnel

### 9.1 Ingress Rule

Added to `/etc/cloudflared/config.yml`:

```yaml
ingress:
  # ... other rules ...
  - hostname: fdroid.fishcmus.io.vn
    service: http://localhost:8083
  - service: http_status:404    # catch-all must be last
```

### 9.2 DNS Route

```bash
cloudflared tunnel route dns <TUNNEL_ID> fdroid.fishcmus.io.vn
```

This creates a CNAME record in Cloudflare DNS pointing `fdroid.fishcmus.io.vn` to the tunnel.

### 9.3 Restart

```bash
sudo systemctl restart cloudflared
```

### 9.4 Verify

```bash
curl -sI https://fdroid.fishcmus.io.vn/repo/index.html
# Should return HTTP/2 200
```

## 10. Publishing Updates

When releasing a new version:

### Step 1: Bump version in `android/app/build.gradle.kts`

```kotlin
defaultConfig {
    versionCode = 2          // increment
    versionName = "1.1"      // update
}
```

### Step 2: Add changelog

```bash
echo "Changelog text here" > android/fastlane/metadata/android/en-US/changelogs/2.txt
```

### Step 3: Build

```bash
cd android && ./gradlew assembleRelease
```

### Step 4: Copy APK and regenerate

```bash
cp android/app/build/outputs/apk/release/app-release.apk /var/www/fdroid/repo/
cd /var/www/fdroid
/var/www/fdroid/.venv/bin/fdroid update
cp /path/to/brand/logo/mascot-512.png /var/www/fdroid/repo/icons/icon.png
```

### Step 5: Update metadata if needed

Edit `/var/www/fdroid/metadata/com.viecz.vieczandroid.yml` and set `CurrentVersionCode` to the new value.

### Step 6: Tag and push

```bash
git tag v1.1
git push origin master --tags
```

## 11. Troubleshooting

### `ResParserError: res1 must be zero!`

The system fdroidserver (apt) has an outdated androguard. Use the venv version:

```bash
/var/www/fdroid/.venv/bin/fdroid update
```

### Repo icon is a grey placeholder

`fdroid update` overwrites `repo/icons/icon.png`. Re-copy after every update:

```bash
cp brand/logo/mascot-512.png /var/www/fdroid/repo/icons/icon.png
```

### F-Droid client shows "index is too old"

Regenerate the index:

```bash
cd /var/www/fdroid && /var/www/fdroid/.venv/bin/fdroid update
```

### cloudflared tunnel not routing

```bash
# Check service status
sudo systemctl status cloudflared

# Verify DNS route exists
cloudflared tunnel route dns <TUNNEL_ID> fdroid.fishcmus.io.vn

# Check config syntax
sudo cat /etc/cloudflared/config.yml
```

### nginx returns 404

```bash
# Verify root directory
ls /var/www/fdroid/repo/

# Test locally
curl http://localhost:8083/repo/index.html
```
