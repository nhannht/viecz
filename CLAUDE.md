## Serena + JetBrains Is the Default
Serena is the primary tool for ALL code work. `Read`/`Grep`/`Glob`/`Edit` only for non-code files (.md, .yml, .json).
- Workflow: overview → drill down → trace usage → check hierarchy → edit
- `Edit` OK for surgical 3-5 line edits inside large functions
- **NEVER** delegate code edits to sub-agents with `isolation: "worktree"`

## Dev Commands

**Android**: `./gradlew assembleDevDebug | installDevDebug | testDevDebugUnitTest`. Flavors: `dev`/`prod`. Always `adb reverse tcp:9999 tcp:9999`.

**Go**: `go test ./...` | `go run cmd/server/main.go`. Test DB: `docker compose -f docker-compose.testdb.yml up -d` (PG 5433, Meili 7700).

**Web**: `bunx ng test` (NOT `bunx vitest run`) | `bun run build`
**NEVER run `bun start`, `ng serve`, `bun run build`, or `ng build` from Claude.** Check build via `tmux capture-pane -t <pane> -p -S -15`.

**Linting**: `cd server && golangci-lint run ./...` | `cd web && bunx eslint 'src/**/*.ts'`

## Learned Patterns
- **PayOS**: Returns `200 OK` before signature check for test payloads (`"VQRIO123"`).
- **Web components**: Production uses `app-task-card` (shared/), NOT `nhannht-metro-*` (Storybook only).
- **Metro theming**: 5 CSS vars via Tailwind. Global theme = `.dracula` on `<html>` (`ThemeService`).
- **Pre-commit hook**: `/code-review` skill runs before `git commit` — enforced in `.claude/settings.json`.
- **golangci-lint v2**: Config uses `linters.settings` (not top-level). Config at `server/.golangci.yml`.
- **Jellyfish submodule**: Serena/JetBrains cannot index git submodules — use `Read`/`Edit` directly.
- **Three.js GLB**: Inspect GLB header before rendering. See `docs/human_docs/by-claude-threejs-learned-patterns.md`.
