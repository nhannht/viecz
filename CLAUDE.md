## ⛔ HARD BLOCKER — YouTrack First (DO NOT SKIP)
**BEFORE writing, editing, or generating ANY code**, you MUST:
1. **Search YouTrack** for an existing issue (`project:KNS #Unresolved` or `project:CVP #Unresolved`)
2. **Create an issue** if none exists (`mcp__youtrack__create_issue`)
3. **Move it to "In Progress"** (`mcp__youtrack__update_issue` with `customFields: {"State": "In Progress"}`)
4. **Only then** start coding

After completing work:
5. **Comment results** on the issue (`mcp__youtrack__add_issue_comment`)
6. **Set state to "Fixed"** (`mcp__youtrack__update_issue` with `customFields: {"State": "Fixed"}`)
7. **Log time** (`mcp__youtrack__log_work`)

**No exceptions.** If the user gives you a coding task and you start writing code without a YouTrack issue, you are violating this rule. This applies to features, bugs, refactors, and any non-trivial change.

## Learned Patterns
- **PayOS**: Returns `200 OK` before signature check for test payloads (`"VQRIO123"`). Client controls `return_url`.
- **Web components**: Production uses `app-task-card` (shared/), NOT `nhannht-metro-*` (Storybook only).
- **Storybook**: All `@storybook/*` packages same major version. `@angular/platform-browser-dynamic` required.
- **Storybook theming**: Manager theme switching via `channel.on('globalsUpdated', ...)` in `manager.ts`. `storybook/core-events` doesn't resolve in manager bundle — use the string `'globalsUpdated'` directly. Preview theme via `componentWrapperDecorator` with `globals['theme']`.
- **Metro theming**: All metro components use 5 CSS vars (`--color-bg`, `--color-fg`, `--color-muted`, `--color-border`, `--color-card`) via Tailwind. Global theme = `.dracula` class on `<html>` (managed by `ThemeService`). Report component has scoped theme system with extra `--color-accent` — doesn't inherit from global.
- **Shell frosted glass**: Use `color-mix(in srgb, var(--color-bg) 85%, transparent)` instead of hardcoded `rgba()` for theme-aware transparency.
- **Claudtest**: Batch ADB commands, minimize screenshots, visual estimation first.
- **Pre-commit hook**: `/code-review` skill MUST run before `git commit` — enforced by `PreToolUse:Bash` hook in `.claude/settings.json`. Commit will be denied without it.
- **golangci-lint v2**: Config uses `linters.settings` (NOT top-level `linters-settings`). Config at `server/.golangci.yml`.
- **Angular ESLint**: Use `ng add angular-eslint --skip-confirmation` to set up. Config at `web/eslint.config.js`. Test files have relaxed rules.
- **Jellyfish .env**: Has extra fields (youtrack_workflow_*, etc.) — Pydantic Settings needs `extra="ignore"`.
- **Jellyfish dev mode**: `cd jellyfish && uv run uvicorn jellyfish.dev:create_dev_app --factory --reload --host 127.0.0.1 --port 8000`
- **claude -p auth**: No API key needed — reads `~/.claude/.credentials.json`. Systemd unit needs `HOME=/home/ubuntu` set.
- **Jellyfish submodule**: Serena/JetBrains cannot index git submodules — use `Read`/`Edit` tools directly for `jellyfish/` code.
- **Three.js GLB models**: ALWAYS inspect GLB JSON header before writing rendering code (`meshes`, `animations`, `primitives[].mode`, `extensionsUsed`). See `docs/human_docs/by-claude-threejs-glb-guide.md`.
- **GLB multiple animations**: Play ONE animation at a time with `mixer.clipAction(clip).play()`. Playing all simultaneously = broken poses. Use `fadeOut()`/`fadeIn()` to crossfade.
- **GLB extensions**: `KHR_materials_pbrSpecularGlossiness` (common on Sketchfab) was removed from Three.js 0.175+. Convert model first: `npx @gltf-transform/cli metalrough input.glb output.glb`. Without conversion, materials render black.
- **Three.js PBR lighting**: PBR/metallic-roughness models render black without scene lights. Always add ambient + directional lights.
- **Three.js bloom**: `EffectComposer` + `UnrealBloomPass` requires `OutputPass` as final pass for tone mapping to work. Use `composer.render()` not `renderer.render()`.
- **angular-three**: `provideNgtRenderer()` in `app.config.ts` is REQUIRED — without it nothing renders (no error, just blank canvas).
