## Taskwarrior First (MANDATORY)
**BEFORE any code change**: `task project:viecz list` to search ‚Üí `task add` if missing ‚Üí `task <id> start` ‚Üí work ‚Üí `task <id> done`. No exceptions.

### Commands
- **List**: `task project:viecz list` ‚Äî list all open tasks
- **Create**: `task add "title" project:viecz priority:H/M/L +feature/+bug/+task`
- **Start**: `task <id> start` ‚Äî mark task as in-progress
- **Done**: `task <id> done` ‚Äî complete a task
- **Modify**: `task <id> modify <changes>` ‚Äî update task details

### Structure
- `project:viecz` (or `viecz.web`, `viecz.server`, `viecz.android`, `viecz.jellyfish`)
- Tags: `+bug`, `+feature`, `+task`, `+chore`, `+research`
- Use `epic:` and `milestone:` UDAs for grouping

### History
- Old GitHub Issues, YouTrack (KNS-*), and git-bug references in commits are historical

## Serena + JetBrains Is the Default
Serena is the primary tool for ALL code work. `Read`/`Grep`/`Glob`/`Edit` only for non-code files (.md, .yml, .json).
- Workflow: overview ‚Üí drill down ‚Üí trace usage ‚Üí check hierarchy ‚Üí edit
- `Edit` OK for surgical 3-5 line edits inside large functions
- **NEVER** delegate code edits to sub-agents with `isolation: "worktree"`

## Dev Commands

**Android**: `./gradlew assembleDevDebug | installDevDebug | testDevDebugUnitTest`. Flavors: `dev`/`prod`. Always `adb reverse tcp:9999 tcp:9999`.

**Go**: `go test ./...` | `go run cmd/server/main.go`. Test DB: `docker compose -f docker-compose.testdb.yml up -d` (PG 5433, Meili 7700).

**Web**: `bunx ng test` (NOT `bunx vitest run`) | `bun run build`
**NEVER run `bun start`, `ng serve`, `bun run build`, or `ng build` from Claude.** Check build via `zellij action dump-screen /tmp/zellij-pane.txt && cat /tmp/zellij-pane.txt`.
**Browser verification**: User's browser runs on local machine, connected via SSH tunnel to dev server. Use `ssh -L 9222:localhost:9222` to forward Chrome DevTools debug port, then use CDP (Chrome DevTools Protocol) to inspect/screenshot what the browser actually renders ‚Äî don't rely on curl or build output alone.

**Linting**: `cd server && golangci-lint run ./...` | `cd web && bunx eslint 'src/**/*.ts'`

## Learned Patterns
- **PayOS**: Returns `200 OK` before signature check for test payloads (`"VQRIO123"`).
- **Web components**: Production uses `app-task-card` (shared/), NOT `nhannht-metro-*` (Storybook only).
- **Metro theming**: 5 CSS vars via Tailwind. Global theme = `.dracula` on `<html>` (`ThemeService`).
- **Pre-commit hook**: `/code-review` skill runs before `git commit` ‚Äî enforced in `.claude/settings.json`.
- **golangci-lint v2**: Config uses `linters.settings` (not top-level). Config at `server/.golangci.yml`.
- **Jellyfish submodule**: Serena/JetBrains cannot index git submodules ‚Äî use `Read`/`Edit` directly.
- **Three.js GLB**: Inspect GLB header before rendering. See `docs/human_docs/by-claude-threejs-learned-patterns.md`.
